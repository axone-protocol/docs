Release binary‚Äã

You'll find the released binaries on the GitHub repository. These are statically linked builds, so be careful to choose the right binary for your system (i.e. linux/amd64 or linux/arm64). Checksums are also provided if you wish to verify the integrity of the download.

To easily install okp4d from a GitHub release you can use this oneliner script:

curl https://i.jpillora.com/okp4/okp4d@{data.okp4dVersion}! | bash
caution

By principle, take the time to verify the checksum manually.

Docker‚Äã

The okp4d binary can be invoked directly through the following docker image:

docker run -ti --rm okp4/okp4d:v6.0.0 --help
tip

Build once, deploy anywhere: The docker image is a multi-arch build supporting both amd64 and arm64 architectures.

Build from source‚Äã

Prerequisites‚Äã

  • go ‚Äî OKP4 is built using Go version 19+. Install Go 1.19+ following instructions from the official Go documentation.
  • make ‚Äî the project comes with a convenient Makefile so verify that make is properly installed.
  • git ‚Äî in order to clone the project you need to have git installed.

Steps‚Äã

Clone the okp4/okp4d repo:

git clone https://github.com/okp4/okp4d
cd okp4d

Install okp4d using the provided Makefile:

make install

You can verify that everything is OK:

okp4d version

You should get an output similar to:

v6.0.0
